12  Chapter 10 — Agents

12.1 Introduction

As AI systems evolve beyond simple request–response interactions, a new architectural component emerges: the agent.

In earlier chapters, we introduced the pipeline as the fundamental structure for organizing information flow. Pipelines transform inputs through a sequence of operations, producing outputs through structured stages. However, pipelines are primarily linear structures. They execute predefined transformations but do not actively decide how those transformations should unfold.

Agents introduce decision-making into the system.

An agent is a computational entity capable of observing its environment, reasoning about its current state, and taking actions that influence future outcomes. Rather than executing a single pass through a pipeline, an agent operates within an iterative loop that allows it to adapt its behavior over time.

At the core of agent architecture lies a simple conceptual cycle.

12.2 The Observe-Think-Act Loop

12.2.1 The Fundamental Cycle

observe → think → act

This loop defines how an agent interacts with its environment. During the observation stage, the agent gathers information from the system state, user input, or external data sources. In the thinking stage, the agent uses a language model or other reasoning mechanism to interpret the situation and determine the next step. During the action stage, the agent executes a command—often invoking a tool, retrieving additional data, or updating the system state.

The result of the action becomes part of the environment that the agent observes in the next iteration. Through this cycle, the agent progressively moves toward completing its assigned objective.

12.2.2 Loop Execution

The observe-think-act loop is not executed once—it repeats until the agent determines that its objective has been achieved or that further progress is impossible. Each iteration refines the agent’s understanding and brings it closer to its goal.

Consider a simple example: an agent tasked with answering a complex research question.

  1. Observe: The agent receives the question and initial context
  2. Think: The agent analyzes the question and identifies what information is needed
  3. Act: The agent searches for relevant documents
  4. Observe: The agent receives the search results
  5. Think: The agent reads the documents and extracts key information
  6. Act: The agent synthesizes the information into a preliminary answer
  7. Observe: The agent reviews the answer for completeness
  8. Think: The agent identifies gaps or inconsistencies
  9. Act: The agent conducts additional searches or refines the answer
  10. Observe: The agent reviews the refined answer
  11. Think: The agent determines the answer is complete
  12. Act: The agent returns the final answer

This iterative process enables the agent to handle complexity that would be difficult or impossible to address in a single pipeline pass.

12.3 Beyond the Basic Loop

12.3.1 Additional Capabilities

While the observe-think-act loop provides the conceptual foundation of agents, practical agent systems incorporate several additional capabilities that enable more sophisticated behavior. Among these capabilities, three are particularly central: planning, tool use, and memory.

12.3.2 Planning

Planning allows the agent to decompose complex tasks into a sequence of intermediate steps. Many problems cannot be solved in a single reasoning pass. Instead, the agent must identify subtasks, determine their order, and execute them systematically.

The Value of Planning

Planning introduces structure into the agent’s behavior, enabling it to approach multi-step problems in a controlled and interpretable manner. Without planning, an agent might pursue goals haphazardly, wasting resources on irrelevant actions or missing critical steps.

Planning in Practice

For example, an agent tasked with answering a complex research question may first identify relevant information sources, retrieve documents, summarize key findings, and then synthesize a final response. Each of these operations becomes part of a structured plan generated during the reasoning phase of the agent loop.

Plan Types

Different planning approaches include:

  • Sequential planning: A linear sequence of steps
  • Hierarchical planning: High-level goals broken into subgoals, each with their own plans
  • Adaptive planning: Plans that are revised based on intermediate results
  • Conditional planning: Plans with branches that depend on conditions

12.3.3 Tool Use

Tool use expands the capabilities of the agent beyond the knowledge embedded within the model itself. While language models can generate text and perform reasoning over internal representations, they cannot directly access external systems or perform specialized computations. Tools provide this bridge between the agent and the external world.

Tool Types

Tools may include:

  • Search engines: For retrieving information from the web
  • Databases: For querying structured or unstructured data
  • APIs: For interacting with external services
  • Calculators: For performing mathematical computations
  • Code interpreters: For executing generated code
  • Domain-specific services: For specialized capabilities

Tool Invocation

During the thinking phase of the loop, the agent may decide to invoke a particular tool in order to gather additional information or perform a computation required for the task. The output of the tool then becomes new information that the agent incorporates into its reasoning.

Tool Selection

Effective agents must decide which tools to use and when. This requires understanding:

  • What each tool can do
  • When each tool is appropriate
  • How to format requests to tools
  • How to interpret tool outputs
  • What to do when tools fail

Tool Chains

Complex tasks may require chaining multiple tools. For example, an agent might:

  1. Search for relevant documents using a search engine
  2. Retrieve the full text of documents using an API
  3. Extract key information using a text processing tool
  4. Perform calculations using a calculator
  5. Format the results using a template tool

Each tool in the chain transforms information, moving the agent closer to its goal.

12.3.4 Memory

Memory provides the final essential component of agent architecture. Unlike stateless pipelines, agents often require the ability to retain information across multiple reasoning cycles. Memory allows the agent to track progress toward a goal, maintain contextual information about previous actions, and accumulate knowledge over time.

Memory Types

Memory may take several forms:

  • Short-term memory: Stores information relevant to the current task, such as intermediate results or partial plans
  • Working memory: Holds information that is actively being used in reasoning
  • Long-term memory: Persists knowledge across sessions, enabling the agent to learn from previous interactions or store structured data that can be retrieved in future tasks
  • Episodic memory: Records specific experiences or episodes that can be referenced later

Memory Operations

Effective memory management requires operations for:

  • Storing: Saving information to memory
  • Retrieving: Accessing stored information
  • Updating: Modifying existing information
  • Forgetting: Removing outdated or irrelevant information

Memory Architecture

Memory can be implemented in various ways:

  • In-memory structures: Simple variables and data structures within the agent’s process
  • Databases: Persistent storage for long-term memory
  • Vector databases: For semantic search and retrieval of memories
  • External storage: Files or cloud storage for large-scale memory

When combined with planning and tool use, memory allows agents to operate as persistent problem-solving entities rather than transient inference calls.

12.4 Agents in System Architecture

12.4.1 Agents and Pipelines

Despite their apparent autonomy, it is important to recognize that agents are still components within larger systems. They do not replace pipelines but extend them.

In many architectures, pipelines handle deterministic processing stages while agents manage dynamic decision-making processes. The two structures complement one another:

  • Pipelines provide stability and predictability
  • Agents introduce flexibility and adaptive reasoning

12.4.2 Hybrid Architectures

Effective AI systems often combine both approaches:

pipeline → agent → pipeline → agent → pipeline

For example, a document analysis system might:

  1. Use a pipeline to preprocess documents (deterministic)
  2. Use an agent to analyze content and extract key information (adaptive)
  3. Use a pipeline to format the extracted information (deterministic)
  4. Use an agent to synthesize findings across documents (adaptive)
  5. Use a pipeline to generate a final report (deterministic)

This hybrid approach leverages the strengths of both structures.

12.4.3 Agent Composition

Complex agents can be composed from simpler agents:

  • Specialist agents: Each agent handles a specific type of task
  • Coordinator agents: Orchestrate the interaction of specialist agents
  • Hierarchical agents: Higher-level agents delegate to lower-level agents

This composition enables sophisticated systems built from reusable agent components.

12.5 Design Considerations

12.5.1 Autonomy vs. Control

Effective agents must be able to reason deeply enough to solve complex problems while remaining constrained by system-level safeguards that ensure reliability. This balance between autonomy and control is one of the central challenges of modern AI engineering.

Autonomy Benefits

  • Flexibility to handle novel situations
  • Ability to pursue goals without constant human guidance
  • Adaptation to changing circumstances

Control Requirements

  • Constraints on allowed actions
  • Mechanisms to override or stop agent behavior
  • Monitoring of agent decisions
  • Validation of agent outputs

12.5.2 Goal Specification

Agents need clear goals. Poorly specified goals lead to unpredictable behavior. Goal specification should include:

  • The objective to achieve
  • Constraints on acceptable methods
  • Criteria for completion
  • Timeout or iteration limits

12.5.3 Error Handling

Agents encounter errors during execution. Robust agents should:

  • Detect when errors occur
  • Attempt to recover from errors
  • Fall back to alternative approaches
  • Report failures clearly
  • Know when to give up

12.5.4 Safety and Alignment

Agents that can take actions in the world require safety mechanisms:

  • Permission boundaries for tool use
  • Approval requirements for high-impact actions
  • Monitoring for potentially harmful behavior
  • Constraints on resource usage

12.6 The Evolution of Intelligence

12.6.1 From Static to Dynamic

The introduction of agents therefore marks a significant step in the evolution of AI systems. Instead of responding to isolated prompts, systems can now pursue objectives through iterative reasoning and action.

This evolution progresses through several stages:

  1. Static inference: Single model call produces a response
  2. Pipelines: Multiple stages transform information
  3. Agents: Iterative loops enable adaptive behavior
  4. Multi-agent systems: Multiple agents collaborate

12.6.2 Operational Intelligence

Through agents, AI systems move closer to a form of operational intelligence. They do not merely generate responses. They interact with their environment, coordinate tools, and pursue goals through structured reasoning processes.

12.6.3 The Engineer’s Evolving Role

The engineer’s task becomes the design of agent architectures that balance autonomy with control. This requires:

  • Understanding agent capabilities and limitations
  • Designing effective goal specifications
  • Implementing robust error handling
  • Providing appropriate safety mechanisms
  • Monitoring and debugging agent behavior

12.7 Conclusion

In this architecture, intelligence is not confined to a single model invocation.

It unfolds across a sequence of observations, decisions, and actions.

Agents represent a fundamental shift in how we think about AI systems. They transform static inference into dynamic system behavior, enabling AI to pursue goals, adapt to circumstances, and operate with increasing autonomy.

The next chapter examines the tools that agents use—the bridges between probabilistic reasoning and deterministic systems. Tools expand the capabilities of agents, allowing them to interact with the external world and perform operations beyond their native intelligence.
