8  Chapter 6 — IDE + Terminal Workflow

8.1 Introduction

Modern development environments offer powerful integrated tools for writing and managing code. Integrated development environments (IDEs) provide advanced capabilities such as code navigation, refactoring, debugging, type analysis, and project management. These features significantly improve the efficiency with which engineers can construct and maintain complex software systems.

However, AI engineering introduces workflows that extend beyond the scope of traditional code editing. Intelligent systems are not composed solely of source files. They involve pipelines that must be executed, agents that must be monitored, models that must be invoked, logs that must be observed, and evaluation processes that must be continuously analyzed.

As a result, the AI engineering workflow requires two complementary environments: the IDE and the terminal.

These environments serve different but equally important roles.

IDE = coding layer
terminal = system layer

8.2 The IDE: Coding Layer

8.2.1 Primary Capabilities

The IDE provides the environment in which the codebase is developed. Its primary capabilities include:

  • Code editing: Advanced editors with syntax highlighting, auto-completion, and intelligent formatting
  • Navigation: Jump to definitions, find references, and search across files
  • Refactoring: Rename symbols, extract functions, and restructure code safely
  • Debugging: Set breakpoints, step through code, and inspect variables
  • Static analysis: Detect errors, suggest improvements, and enforce code quality
  • Project management: Organize files, manage build systems, and run tests

8.2.2 Focus: Internal Structure

Within the IDE, engineers focus primarily on the internal structure of the system:

  • Implementing functions that interact with language models
  • Constructing data structures for model inputs and outputs
  • Designing classes and modules for system organization
  • Writing integration code for tools and APIs
  • Implementing evaluation and validation logic
  • Building infrastructure components and utilities

The IDE provides visibility into the architecture of the codebase and allows engineers to reason about the relationships between components.

8.2.3 When the IDE Excels

The IDE is most valuable for tasks that involve:

  • Complex codebases: Navigating large projects with thousands of files
  • Precise editing: Making targeted changes while maintaining correctness
  • Refactoring: Restructuring code without introducing errors
  • Type safety: Ensuring type correctness in statically typed languages
  • Code understanding: Comprehending how code is organized and how components interact

These tasks benefit from the IDE’s sophisticated analysis and editing capabilities.

8.3 The Terminal: System Layer

8.3.1 Primary Capabilities

The terminal provides the environment in which the system operates. Its primary capabilities include:

  • Process control: Start, stop, monitor, and manage running processes
  • Log observation: Stream and analyze logs in real time
  • Command execution: Run scripts, launch experiments, and execute system commands
  • Session management: Maintain long-running sessions with tmux or similar tools
  • Data inspection: View and manipulate files and data directly
  • System monitoring: Track resource usage and system health

8.3.2 Focus: Operational Behavior

The terminal focuses on how the system behaves when it runs:

  • Launching pipelines and observing their execution
  • Monitoring agent behavior as it executes reasoning steps
  • Streaming logs from multiple concurrent processes
  • Running evaluation scripts and analyzing results
  • Invoking model APIs and observing responses
  • Managing services and infrastructure components

The terminal provides direct access to the operational signals that reveal how the system behaves in practice.

8.3.3 When the Terminal Excels

The terminal is most valuable for tasks that involve:

  • Running experiments: Launching and monitoring iterative experimentation
  • Observing behavior: Watching systems execute in real time
  • Managing processes: Controlling multiple concurrent activities
  • Data manipulation: Processing and transforming data directly
  • Automation: Running scripts and workflows
  • System operations: Interacting with services, databases, and infrastructure

These tasks benefit from the terminal’s direct control and visibility.

8.4 Complementary Strengths

8.4.1 The IDE’s Strengths

The IDE excels at:

  • Structural understanding: Comprehending code organization and dependencies
  • Precise editing: Making changes while maintaining correctness
  • Static analysis: Detecting problems before execution
  • Code quality: Enforcing standards and best practices
  • Developer productivity: Accelerating common development tasks

8.4.2 The Terminal’s Strengths

The terminal excels at:

  • Dynamic understanding: Observing how code behaves when it runs
  • Process control: Managing execution and system state
  • Real-time feedback: Seeing results immediately
  • Workflow automation: Scripting repetitive tasks
  • System integration: Connecting to external services and infrastructure

8.4.3 Why Both Are Necessary

Neither environment alone is sufficient for AI engineering:

  • IDE alone: Engineers can write code but cannot effectively observe how it behaves when integrated with models, tools, and data pipelines
  • Terminal alone: Engineers can run systems but lack the sophisticated editing and analysis capabilities needed for complex codebases

Together, they form a complete development environment that supports both the structural and operational aspects of AI systems construction.

8.5 The Development Cycle

8.5.1 A Four-Stage Loop

The AI engineering workflow can be summarized as a four-stage loop:

write → run → observe → iterate

This cycle is repeated continuously as systems are developed and refined.

8.5.2 Stage 1: Write

The process begins with writing. Within the IDE, the engineer modifies the codebase:

  • Adjusting prompts and prompt templates
  • Implementing or modifying pipeline stages
  • Configuring agent behavior and tool sets
  • Writing evaluation scripts and test cases
  • Updating infrastructure configurations

Structural changes are implemented and verified through local testing tools provided by the development environment. The IDE’s static analysis and code navigation capabilities help ensure correctness during this stage.

8.5.3 Stage 2: Run

Next comes execution. Using the terminal, the engineer launches the relevant system component:

  • Running a script that executes a pipeline
  • Initiating an agent loop to solve a problem
  • Executing a data processing pipeline
  • Starting a local model server
  • Launching an evaluation script on a test dataset

The terminal’s process control capabilities allow engineers to start, monitor, and manage these executions effectively.

8.5.4 Stage 3: Observe

Once the system is running, observation becomes critical. Engineers must:

  • Examine logs to understand the flow of information through the system
  • Inspect outputs to verify correctness and quality
  • Trace how requests propagate through pipelines and agents
  • Monitor metrics to assess performance and resource usage
  • Identify errors, anomalies, or unexpected behaviors

The terminal provides direct access to these signals, allowing engineers to understand how the system behaves under real conditions.

8.5.5 Stage 4: Iterate

Finally, iteration occurs. Insights gained from observation lead to modifications in the codebase:

  • Refine prompts based on observed model behavior
  • Adjust pipeline logic to correct errors or improve efficiency
  • Modify agent strategies based on reasoning patterns
  • Update evaluation criteria to better measure desired behavior
  • Tune infrastructure parameters for improved performance

The engineer returns to the IDE to implement these changes, and the cycle repeats.

8.5.6 The Nature of Iteration

This workflow reflects the experimental nature of AI engineering. Unlike purely deterministic systems, AI systems often require iterative refinement:

  • Models may produce different outputs across runs
  • Prompts may require tuning to achieve desired behavior
  • Pipelines may need adjustment to handle edge cases
  • Agents may require refinement to reason more effectively
  • Evaluation may reveal unexpected failure modes

The IDE and terminal together enable rapid iteration. The IDE facilitates precise modifications, while the terminal enables quick testing and observation.

8.6 The Cooperative Workflow

8.6.1 Seamless Transitions

Effective AI engineering requires seamless transitions between IDE and terminal:

  • From editing a prompt in the IDE to testing it in the terminal
  • From observing an error in logs to fixing it in the IDE
  • From implementing a new feature in the IDE to integrating it in a pipeline
  • From analyzing evaluation results in the terminal to refining the approach in the IDE

Engineers should be able to move between these environments without breaking their flow or losing context.

8.6.2 Shared Context

The IDE and terminal should share context:

  • Working in the same project directory
  • Using the same file system and paths
  • Sharing environment variables and configuration
  • Coordinating on Git operations and version control

This shared context ensures that changes made in one environment are immediately visible in the other.

8.6.3 Complementary Views

The two environments provide complementary views of the system:

  • IDE view: Static structure, relationships, code organization
  • Terminal view: Dynamic behavior, execution flow, operational state

Together, these views provide a complete picture of both how the system is designed and how it behaves in practice.

8.7 Practical Integration

8.7.1 IDE with Integrated Terminal

Many modern IDEs provide integrated terminals that combine both environments in a single interface:

  • Split panes showing code editor and terminal side by side
  • Multiple terminal tabs for different tasks
  • Quick access to terminal commands from within the IDE
  • Synchronization of working directories and projects

This integration reduces context switching and improves workflow efficiency.

8.7.2 Terminal with Editor Integration

Command-line environments can also integrate with editors:

  • Using terminal-based editors like vim or neovim
  • Launching graphical editors from the terminal
  • Using tools that bridge between terminal and GUI
  • Configuring the terminal to invoke the IDE for specific files

This approach allows engineers to stay primarily in the terminal while accessing editor capabilities when needed.

8.7.3 The Right Tool for the Task

Regardless of the specific integration approach, the principle is the same: use the right tool for the task:

  • Use the IDE for tasks that benefit from sophisticated code analysis and editing
  • Use the terminal for tasks that benefit from direct control and visibility
  • Move fluidly between environments as the workflow demands

The goal is not to choose one environment over the other, but to use each for what it does best.

8.8 Scaling the Workflow

8.8.1 Team Collaboration

The IDE and terminal workflow scales to team environments:

  • Code reviews: Conducted in the IDE or code review platforms
  • Shared terminals: For collaborative debugging and system operation
  • Development environments: Standardized across team members
  • Documentation: Captures both structural design and operational procedures

8.8.2 Version Control Integration

Both environments integrate with version control systems:

  • IDE: Git integration for committing, branching, and merging
  • Terminal: Git commands for advanced operations and automation
  • Coordination: Ensuring changes are properly tracked and synchronized

8.8.3 Continuous Integration

The workflow extends to CI/CD pipelines:

  • IDE: Code that will be tested and deployed through CI
  • Terminal: Scripts and configurations for CI/CD processes
  • Integration: Local testing that matches CI/CD behavior

8.9 Conclusion

The IDE and terminal form a cooperative workflow that enables effective AI engineering.

The IDE constructs the system—providing the tools for writing, organizing, and understanding code.

The terminal brings it to life—providing the control and visibility needed to run, observe, and refine systems.

The AI engineer moves continuously between these layers: designing structures in the IDE and operating systems through the terminal. Code defines the architecture of the system, while the terminal reveals how that architecture behaves in practice.

Through this interplay, intelligent systems are gradually refined.

The IDE provides the precision and analysis needed for sophisticated code construction.

The terminal provides the control and observation needed for system operation and experimentation.

Together, they form a unified development environment capable of supporting the full complexity of AI engineering.
