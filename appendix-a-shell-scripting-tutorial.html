35  Appendix A — Shell Scripting Tutorial

35.1 Introduction

Many developers think shell scripts are simply collections of commands. This is misleading.

A shell script is actually a program written in a command-oriented language designed to control processes and automate workflows.

Unlike general-purpose languages such as Python or Java, the shell language is primarily built around command execution and composition.

Understanding shell scripting requires seeing it as a language with: - Grammar - Control flow - Variables - Functions - Pipelines - Process management

35.2 The Execution Model

Every shell script follows the same execution model:

Script
   ↓
Shell Interpreter
   ↓
Command Parsing
   ↓
Process Execution

Example script:

#!/bin/bash

echo "Hello"
date

Execution steps: 1. The shell reads the script line by line 2. Each line is parsed 3. Commands are executed sequentially

35.3 The Shebang

The first line often looks like this:

#!/bin/bash

This is called the shebang. It tells the operating system which interpreter should run the script.

Examples:

#!/bin/bash
#!/usr/bin/env python
#!/usr/bin/env node

Without the shebang, the system may not know how to execute the file.

35.4 The Grammar of Shell Scripts

Shell syntax revolves around commands and arguments. General form:

command arg1 arg2 arg3

Example:

ls -l /home

Structure:

command = ls
argument = -l
argument = /home

The shell splits input into tokens based on whitespace.

35.5 Variables

Variables store values that can be reused.

Example:

name="Alice"
echo $name

Important rule:

NO spaces around =

Correct:

name="Alice"

Incorrect:

name = "Alice"

To read a variable:

echo $name

or

echo ${name}

35.6 Command Substitution

Shell allows capturing the output of commands.

Example:

today=$(date)
echo $today

Execution:

date → output string → stored in variable

Older syntax:

today=`date`

But modern shells prefer $(...).

35.7 Control Flow

35.7.1 If Statements

Basic structure:

if condition
then
    command
fi

Example:

if [ -f file.txt ]
then
    echo "File exists"
fi

Here -f means “file exists.”

Other common tests:

-d  directory exists
-e  file exists
-z  string empty
-n  string not empty

35.7.2 If-Else

Example:

if [ -f file.txt ]
then
    echo "File exists"
else
    echo "File missing"
fi

35.8 Loops

35.8.1 For Loop

Example:

for file in *.txt
do
    echo $file
done

Execution:

find all .txt files
iterate over each file

35.8.2 While Loop

Example:

count=1

while [ $count -le 5 ]
do
    echo $count
    count=$((count+1))
done

35.9 Functions

Shell scripts can define functions.

Example:

greet() {
    echo "Hello $1"
}

Calling:

greet Alice

Output:

Hello Alice

Arguments:

$1 first argument
$2 second argument

35.10 Pipelines

Pipelines are one of the most powerful features of the shell.

Example:

ps aux | grep python

Execution model:

ps aux
   ↓
output pipe
   ↓
grep python

Each program reads input from the previous program. This design allows program composition.

35.11 Redirection

Shell also supports redirecting input and output.

Example:

echo "hello" > file.txt

Meaning:

write output to file

Append:

echo "hello" >> file.txt

Input redirection:

sort < file.txt

35.12 Process Control

Shell scripts often manage processes.

Running a command in the background:

python server.py &

The shell returns immediately.

List background jobs:

jobs

Bring a job to foreground:

fg

35.13 The Shell Programming Model

We can summarize shell scripting as a process orchestration language.

Variables
   ↓
Commands
   ↓
Pipelines
   ↓
Processes
   ↓
Output

Unlike traditional programming languages that focus on algorithms, shell scripts focus on coordinating programs.

35.14 The Conceptual Model

Shell scripts operate as a control layer above the operating system.

Script
   ↓
Shell
   ↓
Programs
   ↓
Operating System

The shell acts as a workflow controller.

35.15 Conclusion

Shell scripting is best understood not as a traditional programming language but as a process orchestration language.

Its strengths lie in: - Combining tools - Automating workflows - Managing processes - Interacting with the operating system

By mastering shell scripting, a developer gains direct control over the underlying system.
