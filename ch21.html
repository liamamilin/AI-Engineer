26  Chapter 21 — Building Your First AI System

26.1 Introduction

The preceding chapters have established the theoretical foundations of AI engineering: the shift from deterministic to probabilistic systems, the layered architecture of the AI engineering stack, and the principles of systems thinking, pipelines, agents, and evaluation.

Theory alone, however, is insufficient for building practical AI systems. Concepts become capabilities only through implementation.

This chapter bridges theory and practice. It guides you through building your first complete AI system from scratch—a question-answering application that integrates models, prompts, tools, and evaluation into a functioning system.

26.2 The Project: A Knowledge Assistant

26.2.1 Project Overview

We will build a knowledge assistant that can:

  • Answer questions about a specific domain
  • Retrieve relevant information from a knowledge base
  • Generate accurate, well-structured responses
  • Evaluate its own answers for quality

This project demonstrates the core components of the AI engineering stack in action:

  • Model Layer: Using a language model for reasoning
  • Control Layer: Designing prompts and integrating tools
  • System Layer: Building pipelines and evaluation mechanisms

26.2.2 System Architecture

The system follows a retrieval-augmented generation pattern:

user query
    ↓
preprocessing
    ↓
retrieval (vector database)
    ↓
prompt construction
    ↓
model inference
    ↓
answer generation
    ↓
evaluation
    ↓
response

This architecture is simple yet realistic. It represents the structure of many production AI systems while remaining implementable in a single session.

26.3 Step 1: Project Setup

Create a clean, organized project structure:

cd knowledge-assistant

The directory structure separates concerns:

  • src/: Source code for deterministic components
  • configs/: Configuration files
  • prompts/: Prompt templates
  • eval/: Evaluation datasets and results
  • tests/: Test files
  • docs/: Documentation
  • logs/: System logs

26.4 Step 2: The Model Client

26.4.1 Basic Model Invocation

First, create a simple model client that can invoke the language model:

# src/model_client.py
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()

class ModelClient:
    def __init__(self, model_name="gpt-3.5-turbo"):
        self.client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
        self.model_name = model_name
    
    def generate(self, prompt, temperature=0.7):
        """Generate a response from the model."""
        response = self.client.chat.completions.create(
            model=self.model_name,
            messages=[{"role": "user", "content": prompt}],
            temperature=temperature
        )
        return response.choices[0].message.content
    
    def generate_with_system(self, system_prompt, user_prompt, temperature=0.7):
        """Generate a response with a system prompt."""
        response = self.client.chat.completions.create(
            model=self.model_name,
            messages=[
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": user_prompt}
            ],
            temperature=temperature
        )
        return response.choices[0].message.content

26.5 Step 3: Prompt Design

26.5.1 The System Prompt

Create a system prompt that defines the assistant’s behavior:

# prompts/system.txt
You are a knowledgeable assistant specializing in answering questions based on provided context.

Your task is to:
1. Carefully read the user's question
2. Review the provided context information
3. Answer the question using only the information in the context
4. If the context doesn't contain enough information, clearly state that
5. Provide accurate, well-structured answers

Be concise but thorough. Use clear language and avoid speculation.

26.5.2 The Answer Prompt Template

Create a template for constructing prompts:

# prompts/answer_template.txt
Context:
{context}

Question:
{question}

Please answer the question based on the provided context.

26.5.3 Loading Prompts

Create a utility to load prompts:

# src/prompt_loader.py
import os

def load_prompt(prompt_name):
    """Load a prompt from the prompts directory."""
    prompt_path = os.path.join("prompts", f"{prompt_name}.txt")
    with open(prompt_path, "r") as f:
        return f.read().strip()

def format_template(template, **kwargs):
    """Format a prompt template with provided variables."""
    return template.format(**kwargs)

26.6 Step 4: Building the Pipeline

26.6.1 Pipeline Architecture

Create a pipeline that orchestrates the components:

# src/pipeline.py
from src.model_client import ModelClient
from src.prompt_loader import load_prompt, format_template

class KnowledgePipeline:
    def __init__(self):
        self.model_client = ModelClient()
        self.system_prompt = load_prompt("system")
        self.answer_template = load_prompt("answer_template")
    
    def preprocess(self, query):
        """Preprocess the user query."""
        return query.strip().lower()
    
    def construct_prompt(self, query, context=""):
        """Construct the prompt for the model."""
        user_prompt = format_template(
            self.answer_template,
            context=context,
            question=query
        )
        return user_prompt
    
    def generate_answer(self, query, context=""):
        """Generate an answer for the query."""
        user_prompt = self.construct_prompt(query, context)
        answer = self.model_client.generate_with_system(
            self.system_prompt,
            user_prompt,
            temperature=0.3  # Lower temperature for more factual answers
        )
        return answer
    
    def run(self, query, context=""):
        """Run the complete pipeline."""
        # Preprocess
        cleaned_query = self.preprocess(query)
        
        # Generate answer
        answer = self.generate_answer(cleaned_query, context)
        
        return answer

26.7.2 Creating a Knowledge Base

# src/knowledge_base.py
import chromadb
from chromadb.config import Settings

class KnowledgeBase:
    def __init__(self, collection_name="knowledge"):
        self.client = chromadb.Client(Settings())
        self.collection = self.client.get_or_create_collection(
            name=collection_name
        )
    
    def add_documents(self, documents, metadatas=None):
        """Add documents to the knowledge base."""
        self.collection.add(
            documents=documents,
            metadatas=metadatas or [{}] * len(documents),
            ids=[f"doc_{i}" for i in range(len(documents))]
        )
    
    def query(self, query_text, n_results=3):
        """Query the knowledge base."""
        results = self.collection.query(
            query_texts=[query_text],
            n_results=n_results
        )
        return results["documents"][0]

26.7.4 Integrating Retrieval into the Pipeline

# src/pipeline.py (updated)
from src.model_client import ModelClient
from src.prompt_loader import load_prompt, format_template
from src.knowledge_base import KnowledgeBase

class KnowledgePipeline:
    def __init__(self):
        self.model_client = ModelClient()
        self.system_prompt = load_prompt("system")
        self.answer_template = load_prompt("answer_template")
        self.knowledge_base = KnowledgeBase()
    
    def retrieve_context(self, query, n_results=3):
        """Retrieve relevant context from the knowledge base."""
        documents = self.knowledge_base.query(query, n_results)
        return "\n\n".join(documents)
    
    def preprocess(self, query):
        """Preprocess the user query."""
        return query.strip()
    
    def construct_prompt(self, query, context):
        """Construct the prompt for the model."""
        user_prompt = format_template(
            self.answer_template,
            context=context,
            question=query
        )
        return user_prompt
    
    def generate_answer(self, query, context):
        """Generate an answer for the query."""
        user_prompt = self.construct_prompt(query, context)
        answer = self.model_client.generate_with_system(
            self.system_prompt,
            user_prompt,
            temperature=0.3
        )
        return answer
    
    def run(self, query):
        """Run the complete pipeline with retrieval."""
        # Preprocess
        cleaned_query = self.preprocess(query)
        
        # Retrieve context
        context = self.retrieve_context(cleaned_query)
        
        # Generate answer
        answer = self.generate_answer(cleaned_query, context)
        
        return answer

26.8 Step 6: Adding Evaluation

26.8.1 Simple Evaluation Metrics

# src/evaluator.py
class Evaluator:
    def __init__(self):
        self.results = []
    
    def evaluate_answer(self, query, answer, expected_keywords=None):
        """Evaluate an answer."""
        result = {
            "query": query,
            "answer": answer,
            "has_keywords": True,
            "length": len(answer),
            "timestamp": datetime.now()
        }
        
        # Check for expected keywords
        if expected_keywords:
            result["has_keywords"] = all(
                keyword.lower() in answer.lower()
                for keyword in expected_keywords
            )
        
        self.results.append(result)
        return result
    
    def get_summary(self):
        """Get evaluation summary."""
        if not self.results:
            return {}
        
        total = len(self.results)
        passed = sum(1 for r in self.results if r["has_keywords"])
        
        return {
            "total_evaluations": total,
            "passed": passed,
            "failed": total - passed,
            "pass_rate": passed / total if total > 0 else 0
        }

26.8.2 Running Evaluations

# tests/test_system.py
from src.pipeline import KnowledgePipeline
from src.evaluator import Evaluator

def test_system():
    pipeline = KnowledgePipeline()
    evaluator = Evaluator()
    
    # Test cases
    test_cases = [
        {
            "query": "Who created Python?",
            "expected_keywords": ["Guido", "Rossum"]
        },
        {
            "query": "When was Python first released?",
            "expected_keywords": ["1991"]
        },
        {
            "query": "What programming paradigms does Python support?",
            "expected_keywords": ["procedural", "object-oriented", "functional"]
        }
    ]
    
    for test_case in test_cases:
        answer = pipeline.run(test_case["query"])
        print(f"\nQuery: {test_case['query']}")
        print(f"Answer: {answer}")
        
        evaluator.evaluate_answer(
            test_case["query"],
            answer,
            test_case["expected_keywords"]
        )
    
    # Print summary
    summary = evaluator.get_summary()
    print(f"\nEvaluation Summary:")
    print(f"Total: {summary['total_evaluations']}")
    print(f"Passed: {summary['passed']}")
    print(f"Failed: {summary['failed']}")
    print(f"Pass Rate: {summary['pass_rate']:.2%}")

if __name__ == "__main__":
    test_system()

26.9 Step 7: Creating a Command-Line Interface

# main.py
import argparse
from src.pipeline import KnowledgePipeline
from src.evaluator import Evaluator

def interactive_mode():
    """Run the system in interactive mode."""
    pipeline = KnowledgePipeline()
    print("Knowledge Assistant (type 'quit' to exit)")
    print("-" * 50)
    
    while True:
        query = input("\nYour question: ").strip()
        
        if query.lower() in ['quit', 'exit', 'q']:
            print("Goodbye!")
            break
        
        if not query:
            continue
        
        answer = pipeline.run(query)
        print(f"\nAnswer: {answer}")

def evaluation_mode():
    """Run the system in evaluation mode."""
    from tests.test_system import test_system
    test_system()

def main():
    parser = argparse.ArgumentParser(description="Knowledge Assistant")
    parser.add_argument(
        "mode",
        choices=["interactive", "evaluate"],
        help="Run mode: interactive or evaluate"
    )
    
    args = parser.parse_args()
    
    if args.mode == "interactive":
        interactive_mode()
    elif args.mode == "evaluate":
        evaluation_mode()

if __name__ == "__main__":
    main()

26.10 Step 8: Production Considerations

26.10.1 Adding Logging

# src/logger.py
import logging
from datetime import datetime

def setup_logger(name="knowledge_assistant"):
    """Set up logging for the application."""
    logger = logging.getLogger(name)
    logger.setLevel(logging.INFO)
    
    # Create logs directory if it doesn't exist
    os.makedirs("logs", exist_ok=True)
    
    # File handler
    log_file = f"logs/{datetime.now().strftime('%Y%m%d')}.log"
    file_handler = logging.FileHandler(log_file)
    file_handler.setLevel(logging.INFO)
    
    # Console handler
    console_handler = logging.StreamHandler()
    console_handler.setLevel(logging.INFO)
    
    # Formatter
    formatter = logging.Formatter(
        '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
    )
    file_handler.setFormatter(formatter)
    console_handler.setFormatter(formatter)
    
    logger.addHandler(file_handler)
    logger.addHandler(console_handler)
    
    return logger

26.10.2 Adding Error Handling

# src/pipeline.py (updated with error handling)
class KnowledgePipeline:
    # ... (previous code)
    
    def run(self, query, max_retries=2):
        """Run the complete pipeline with error handling."""
        try:
            # Preprocess
            cleaned_query = self.preprocess(query)
            
            if not cleaned_query:
                return "Please provide a valid question."
            
            # Retrieve context
            context = self.retrieve_context(cleaned_query)
            
            # Generate answer with retry logic
            for attempt in range(max_retries + 1):
                try:
                    answer = self.generate_answer(cleaned_query, context)
                    return answer
                except Exception as e:
                    if attempt < max_retries:
                        logger.warning(
                            f"Attempt {attempt + 1} failed: {e}. Retrying..."
                        )
                        continue
                    else:
                        logger.error(f"All attempts failed: {e}")
                        return "Sorry, I encountered an error generating your answer."
        
        except Exception as e:
            logger.error(f"Pipeline error: {e}")
            return "Sorry, I encountered an error processing your request."

26.11 Best Practices

26.11.1 1. Separate Concerns

  • Keep model logic separate from pipeline logic
  • Separate prompt templates from code
  • Isolate evaluation from production code

26.11.2 2. Design for Observability

  • Log all pipeline stages
  • Track query-answer pairs
  • Monitor system performance

26.11.3 3. Test Continuously

  • Write unit tests for each component
  • Create integration tests for the full pipeline
  • Maintain evaluation datasets

26.11.4 4. Handle Errors Gracefully

  • Implement retry logic for model calls
  • Provide fallback responses
  • Log errors for debugging

26.11.5 5. Optimize for Iteration

  • Keep prompts version-controlled
  • Parameterize configurations
  • Make it easy to test changes

26.12 Common Pitfalls

26.12.1 1. Hard-Coding Values

Problem: Hard-coding API keys, model names, or prompts makes the system difficult to maintain.

Solution: Use configuration files and environment variables.

26.12.2 2. Ignoring Error Handling

Problem: Without error handling, the system crashes on edge cases.

Solution: Implement try-catch blocks and fallback mechanisms.

26.12.3 3. Missing Observability

Problem: Without logs, it’s impossible to debug issues in production.

Solution: Add structured logging at each pipeline stage.

26.12.4 4. Over-Engineering

Problem: Adding too much complexity early makes the system hard to understand.

Solution: Start simple, iterate based on needs.

26.12.5 5. No Evaluation

Problem: Without evaluation, you don’t know if changes improve the system.

Solution: Build evaluation from the start and run it regularly.
