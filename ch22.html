27  Chapter 22 — Practical Prompt Engineering

27.1 Introduction

Chapter 8 established prompt engineering as an architectural discipline—prompts function as interfaces between deterministic systems and probabilistic models. This chapter translates that conceptual framework into practical techniques and patterns.

Effective prompt engineering is not about clever wording. It is a systematic design process that involves understanding tasks, structuring information, specifying constraints, and iteratively refining based on observed behavior.

This chapter provides concrete techniques for designing, testing, and optimizing prompts. It covers patterns, templates, evaluation methods, and best practices that transform prompt crafting from an art into an engineering practice.

27.2 The Prompt Engineering Workflow

27.2.1 Systematic Design Process

Effective prompt engineering follows a structured workflow:

understand task → select pattern → draft prompt → test with examples → analyze failures → refine → iterate

This workflow ensures that prompts are designed systematically rather than through trial and error.

27.2.2 Step 1: Understand the Task

Before writing a prompt, clarify:

  • What is the model supposed to do? (task definition)
  • What information does it need? (required context)
  • What constraints apply? (format, length, style)
  • What does good output look like? (success criteria)

Example: If building a summarization system, define: - Task: Summarize technical documents - Context: Document text, target audience - Constraints: Maximum length, technical level - Success: Covers key points, accurate, readable

27.2.3 Step 2: Select a Pattern

Choose an appropriate prompt pattern based on the task:

  • Structured Instruction: For clear, step-by-step tasks
  • Role-Based: For domain-specific expertise
  • Chain of Thought: For reasoning tasks
  • Few-Shot: For tasks requiring examples
  • Step-by-Step: For multi-step problems

27.2.4 Step 3: Draft the Prompt

Write an initial prompt following the selected pattern. Include:

  • Clear task description
  • Relevant context
  • Explicit constraints
  • Format specifications
  • Examples (if using few-shot)

27.2.5 Step 4: Test with Examples

Evaluate the prompt on representative inputs:

  • Normal cases: Typical inputs
  • Edge cases: Unusual or difficult inputs
  • Negative cases: Inputs that should be rejected

27.2.6 Step 5: Analyze Failures

Identify where the prompt produces poor results:

  • Misunderstanding the task
  • Ignoring constraints
  • Producing incorrect information
  • Failing on edge cases

27.2.7 Step 6: Refine and Iterate

Adjust the prompt based on failure analysis:

  • Clarify instructions
  • Add more examples
  • Strengthen constraints
  • Reorganize structure

27.3 Essential Prompt Patterns

27.3.1 Pattern 1: Structured Instruction

Organize the prompt into clear sections:

Task: [clear description of what to do]

Context: [relevant background information]

Constraints: [limitations on the output]

Format: [required output structure]

Examples: [optional examples]

Output: [the result]

When to Use: General-purpose tasks where clarity is essential.

Example:

Task: Summarize the following technical document.

Context: This is a research paper about machine learning. The summary should be accessible to software engineers who are not machine learning experts.

Constraints:
- Maximum 3 paragraphs
- Include the main contribution
- Use simple language
- Avoid technical jargon when possible

Format: Plain text without bullet points

Document:
[document text]

Output:

Best Practices: - Use clear section headers - Keep each section focused - Be explicit about constraints - Specify output format precisely

27.3.2 Pattern 2: Role-Based

Assign a specific role to the model:

You are an expert [domain specialist] with [specific expertise].

Your role is to [task description].

Please provide a response that demonstrates expertise in [domain].

[additional instructions]

When to Use: Domain-specific tasks requiring specialized knowledge or perspective.

Example:

You are an expert software architect with 20 years of experience designing large-scale distributed systems.

Your role is to review the following system design and provide feedback on scalability, reliability, and maintainability.

Please provide a response that demonstrates expertise in:
- System architecture patterns
- Scalability considerations
- Failure handling strategies
- Best practices for distributed systems

Be thorough but concise. Focus on actionable recommendations.

System Design:
[design description]

Your Review:

Best Practices: - Define the role clearly - Specify the expertise level - Explain what the role entails - Set expectations for the response

27.3.3 Pattern 3: Chain of Thought

Guide the model through explicit reasoning steps:

Let's work through this problem step by step.

Step 1: [first reasoning step]
Step 2: [second reasoning step]
Step 3: [third reasoning step]
...

Based on this reasoning, the answer is: [final answer]

Problem: [the problem to solve]

When to Use: Complex reasoning tasks, math problems, logical deduction.

Example:

Let's work through this database optimization problem step by step.

Step 1: Identify the slow queries by examining the query execution plan.
Step 2: Analyze the table structure and indexes to find missing opportunities.
Step 3: Consider the query patterns and data distribution.
Step 4: Propose index changes based on the analysis.
Step 5: Estimate the performance improvement.

Based on this reasoning, provide your optimization recommendations.

Problem:
A database with 10 million rows is experiencing slow query performance. The main query involves joining three tables on ID columns. Current execution time is 45 seconds.

Optimization Plan:

Best Practices: - Break down complex problems - Make each step explicit - Show the reasoning process - Arrive at a clear conclusion

27.3.4 Pattern 4: Few-Shot Learning

Provide examples within the prompt:

Here are some examples of [task]:

Example 1:
Input: [input 1]
Output: [output 1]

Example 2:
Input: [input 2]
Output: [output 2]

Example 3:
Input: [input 3]
Output: [output 3]

Now, complete this:
Input: [new input]
Output:

When to Use: Tasks where examples demonstrate the desired behavior better than instructions.

Example:

Here are some examples of extracting key information from bug reports:

Example 1:
Input: "When I click the submit button, nothing happens. The console shows 'Error 500'."
Output:
Severity: High
Component: Submit functionality
Error: Error 500

Example 2:
Input: "The text is too small on mobile devices. I can barely read it."
Output:
Severity: Medium
Component: UI/Mobile
Error: Text size issue

Example 3:
Input: "I can't login with my Google account. It just spins forever."
Output:
Severity: High
Component: Authentication
Error: Google login hanging

Now, extract information from this bug report:
Input: "The images load very slowly on the product page. It takes like 10 seconds."
Output:

Best Practices: - Provide diverse examples - Cover edge cases - Show clear input-output relationships - Use consistent formatting

27.3.5 Pattern 5: Step-by-Step Reasoning

Explicitly instruct the model to reason step by step:

Think through this problem step by step:

1. First, [first step]
2. Then, [second step]
3. Next, [third step]
4. Finally, [fourth step]

Problem: [the problem to solve]

Solution:

When to Use: Multi-step problems where the process matters as much as the result.

Example:

Think through this code review step by step:

1. First, examine the overall structure and architecture.
2. Then, check for common security vulnerabilities.
3. Next, analyze the error handling and edge cases.
4. Finally, provide specific recommendations for improvement.

Code to Review:
[code snippet]

Code Review:

Best Practices: - Number the steps clearly - Make each step actionable - Ensure steps build on each other - Lead to a complete solution

27.4 Advanced Prompt Techniques

27.4.1 Technique 1: Parameterized Prompts

Create reusable prompt templates with parameters:

# prompts/summarization_template.txt
Task: Summarize the following text.

Context: {context}

Constraints:
- Maximum length: {max_length}
- Target audience: {audience}
- Tone: {tone}

Format: {format}

Text:
{text}

Summary:
def format_summarization_prompt(text, context="", max_length=200, 
                                audience="general", tone="neutral", 
                                format="plain"):
    """Format the summarization prompt with parameters."""
    template = load_prompt("summarization_template")
    return template.format(
        text=text,
        context=context,
        max_length=max_length,
        audience=audience,
        tone=tone,
        format=format
    )

Benefits: - Reusability across contexts - Easy parameter tuning - Clear what varies - Version control of templates

27.4.2 Technique 2: Chain Prompts

Break complex tasks into sequential prompts:

# Step 1: Extract key points
points_prompt = """
Extract the key points from this text:
{text}

Key points:
"""

# Step 2: Summarize key points
summary_prompt = """
Summarize these key points:
{points}

Summary:
"""

def chain_summarize(text):
    # Step 1
    points = model.generate(points_prompt.format(text=text))
    
    # Step 2
    summary = model.generate(summary_prompt.format(points=points))
    
    return summary

Benefits: - More reliable on complex tasks - Each step can be optimized independently - Easier to debug failures - Can use different models for different steps

27.4.3 Technique 3: Self-Reflection

Ask the model to critique and improve its own output:

def generate_with_reflection(query):
    # Initial generation
    initial_prompt = f"Answer this question: {query}"
    initial_answer = model.generate(initial_prompt)
    
    # Reflection
    reflection_prompt = f"""
Review this answer:
Question: {query}
Answer: {initial_answer}

Critique the answer and identify:
1. Any factual errors
2. Missing information
3. Unclear explanations

Then provide an improved answer.

Critique and Improved Answer:
"""
    
    improved = model.generate(reflection_prompt)
    return improved

Benefits: - Improves answer quality - Reduces hallucinations - Makes reasoning explicit - Provides transparency

27.4.4 Technique 4: Multiple Perspectives

Ask the model to consider different viewpoints:

def analyze_from_perspectives(problem):
    perspectives = [
        "technical perspective",
        "business perspective",
        "user experience perspective"
    ]
    
    analysis = {}
    for perspective in perspectives:
        prompt = f"""
Analyze this problem from a {perspective}:
{problem}

Analysis:
"""
        analysis[perspective] = model.generate(prompt)
    
    # Synthesize perspectives
    synthesis_prompt = f"""
Synthesize these perspectives into a comprehensive analysis:

Technical Analysis:
{analysis['technical perspective']}

Business Analysis:
{analysis['business perspective']}

UX Analysis:
{analysis['user experience perspective']}

Comprehensive Analysis:
"""
    
    return model.generate(synthesis_prompt)

Benefits: - More thorough analysis - Considers multiple stakeholders - Reduces bias - Provides richer insights

27.4.5 Technique 5: Structured Output

Enforce specific output formats:

Format your output as JSON with the following structure:
{
  "name": "string",
  "date": "string (YYYY-MM-DD)",
  "amount": "number",
  "status": "pending|completed|cancelled"
}

27.6 Prompt Evaluation

27.6.1 Automated Evaluation

# src/prompt_evaluator.py
class PromptEvaluator:
    def __init__(self, model):
        self.model = model
    
    def evaluate(self, prompt, test_cases, metrics):
        """Evaluate a prompt against test cases."""
        results = []
        
        for test_case in test_cases:
            output = self.model.generate(prompt.format(**test_case['input']))
            
            result = {
                'input': test_case['input'],
                'expected': test_case['expected'],
                'actual': output,
                'metrics': {}
            }
            
            # Calculate metrics
            for metric in metrics:
                result['metrics'][metric] = self.calculate_metric(
                    metric,
                    test_case['expected'],
                    output
                )
            
            results.append(result)
        
        return self.aggregate_results(results)
    
    def calculate_metric(self, metric, expected, actual):
        """Calculate a specific metric."""
        if metric == 'exact_match':
            return 1 if expected.strip() == actual.strip() else 0
        elif metric == 'keyword_match':
            expected_keywords = set(expected.lower().split())
            actual_keywords = set(actual.lower().split())
            return len(expected_keywords & actual_keywords) / len(expected_keywords)
        elif metric == 'length_appropriate':
            max_length = 200
            return 1 if len(actual) <= max_length else 0
        # ... more metrics
    
    def aggregate_results(self, results):
        """Aggregate results across all test cases."""
        total = len(results)
        metrics = results[0]['metrics'].keys()
        
        aggregated = {}
        for metric in metrics:
            scores = [r['metrics'][metric] for r in results]
            aggregated[metric] = {
                'mean': sum(scores) / total,
                'min': min(scores),
                'max': max(scores)
            }
        
        return aggregated

27.6.2 A/B Testing Prompts

def ab_test_prompts(prompt_a, prompt_b, test_cases):
    """Compare two prompts using A/B testing."""
    results = {
        'prompt_a': evaluate_prompt(prompt_a, test_cases),
        'prompt_b': evaluate_prompt(prompt_b, test_cases)
    }
    
    # Statistical comparison
    comparison = {}
    for metric in results['prompt_a'].keys():
        a_scores = results['prompt_a'][metric]
        b_scores = results['prompt_b'][metric]
        
        comparison[metric] = {
            'a_mean': a_scores['mean'],
            'b_mean': b_scores['mean'],
            'difference': b_scores['mean'] - a_scores['mean'],
            'improvement': (b_scores['mean'] - a_scores['mean']) / a_scores['mean']
        }
    
    return comparison

27.7 Prompt Optimization Techniques

27.7.1 Technique 1: Iterative Refinement

def optimize_prompt(initial_prompt, test_cases, max_iterations=5):
    """Optimize a prompt through iterative refinement."""
    current_prompt = initial_prompt
    best_prompt = initial_prompt
    best_score = 0
    
    for iteration in range(max_iterations):
        # Evaluate current prompt
        score = evaluate_prompt(current_prompt, test_cases)
        
        if score > best_score:
            best_score = score
            best_prompt = current_prompt
        
        # Analyze failures
        failures = analyze_failures(current_prompt, test_cases)
        
        # Generate improvements
        improvements = suggest_improvements(failures)
        
        # Update prompt
        current_prompt = apply_improvements(current_prompt, improvements)
    
    return best_prompt, best_score

27.7.2 Technique 2: Parameter Tuning

def tune_parameters(template, param_ranges, test_cases):
    """Tune prompt parameters."""
    import itertools
    
    # Generate all parameter combinations
    param_names = list(param_ranges.keys())
    param_values = list(itertools.product(*param_ranges.values()))
    
    best_params = None
    best_score = 0
    
    for values in param_values:
        params = dict(zip(param_names, values))
        prompt = template.format(**params)
        score = evaluate_prompt(prompt, test_cases)
        
        if score > best_score:
            best_score = score
            best_params = params
    
    return best_params, best_score

# Example
template = """
Task: Summarize this text.
Max length: {max_length}
Tone: {tone}

Text: {text}
"""

param_ranges = {
    'max_length': [100, 200, 300],
    'tone': ['formal', 'neutral', 'casual']
}

best_params, best_score = tune_parameters(template, param_ranges, test_cases)

27.8 Common Pitfalls and Solutions

27.8.1 Pitfall 1: Over-Specification

Problem: Including too many constraints confuses the model.

Example:

Summarize this text. Use exactly 3 sentences. Start with "The article discusses". End with "In conclusion". Include at least 2 key points. Use formal language. Avoid jargon. Keep sentences under 20 words.

Solution: Prioritize the most important constraints.

Summarize this text in 3 sentences. Include the main points. Use formal language.

Text: {text}

Summary:

27.8.2 Pitfall 2: Ambiguous Instructions

Problem: Vague instructions lead to unpredictable outputs.

Example:

Make this better.

Solution: Be specific about what “better” means.

Improve the readability of this text by:
1. Using shorter sentences
2. Clarifying technical terms
3. Improving paragraph structure

Text: {text}

Improved text:

27.8.3 Pitfall 3: Missing Format Specification

Problem: Without format constraints, outputs vary unpredictably.

Solution: Specify output format explicitly.

Format your answer as JSON:
{
  "summary": "string",
  "key_points": ["string", "string"],
  "confidence": "high|medium|low"
}

Text: {text}

Output:

27.8.4 Pitfall 4: Ignoring Edge Cases

Problem: Prompts work on typical inputs but fail on edge cases.

Solution: Test on diverse inputs and add constraints for edge cases.

Summarize this text.

Special cases:
- If text is empty, return "No content to summarize"
- If text is very short, return it unchanged
- If text is in a different language, translate first

Text: {text}

Summary:

27.8.5 Pitfall 5: Lack of Examples

Problem: Instructions alone don’t convey the desired behavior.

Solution: Use few-shot learning with representative examples.

27.10 Best Practices Summary

27.10.1 Design Principles

  1. Be Specific: Use precise language and explicit constraints
  2. Provide Context: Include relevant background information
  3. Use Patterns: Apply proven prompt patterns for consistency
  4. Test Thoroughly: Evaluate on diverse test cases
  5. Iterate Continuously: Refine based on observed behavior

27.10.2 Development Workflow

  1. Start with a simple prompt
  2. Add complexity incrementally
  3. Test at each step
  4. Analyze failures systematically
  5. Document changes and rationale

27.10.3 Organizational Practices

  1. Build a prompt library for reuse
  2. Version prompts to track changes
  3. Standardize prompt documentation
  4. Share prompts across teams
  5. Establish prompt review processes

27.11 Conclusion

Effective prompt engineering is a systematic practice that combines pattern recognition, clear communication, and iterative refinement. By applying structured patterns, building prompt libraries, and evaluating systematically, engineers can design prompts that reliably elicit desired behavior from language models.

The techniques in this chapter—structured patterns, parameterization, chain prompting, self-reflection, and evaluation—transform prompt engineering from ad hoc experimentation into a disciplined engineering practice.
